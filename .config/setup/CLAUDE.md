# Desktop Setup ‚Äî ~/.config/setup

Bootstraps a fresh Arch Linux + Hyprland desktop after `archinstall`.
Scripts live in `desktop/` (tracked by yadm). Static config files belong in yadm directly ‚Äî scripts only run commands, never write dotfiles.

## File layout

```
~/.config/setup/
‚îú‚îÄ‚îÄ CLAUDE.md
‚îî‚îÄ‚îÄ desktop/
    ‚îú‚îÄ‚îÄ 1-post-install.sh   # Run from archinstall env before reboot ‚Äî LUKS/TPM, GRUB, mkinitcpio
    ‚îú‚îÄ‚îÄ 2-bootstrap.sh      # Entry point ‚Äî calls all scripts/ in order, prints manual steps at end
    ‚îî‚îÄ‚îÄ scripts/

        ‚îú‚îÄ‚îÄ shell.sh        # Oh My Zsh, chsh to zsh
        ‚îú‚îÄ‚îÄ neovim.sh       # Clone kickstart.nvim, install plugins headlessly
        ‚îî‚îÄ‚îÄ fonts.sh        # Write /etc/fonts/local.conf (uses sudo), fc-cache
```

## Bootstrap flow (new machine)

1. Run `archinstall`
2. `bash ~/.config/setup/desktop/1-post-install.sh <device> <luks-uuid>`
3. Reboot
4. `pacman -S yadm git age`
5. `yadm clone https://github.com/Encephala/yadm` (HTTPS ‚Äî no key yet)
6. `yadm decrypt` ‚Äî restores SSH key + SSH config, prompts for age passphrase
7. `yadm remote set-url origin git@github.com:Encephala/yadm`
8. `bash ~/.config/setup/desktop/2-bootstrap.sh`

## yadm

Dotfiles repo: https://github.com/Encephala/yadm

Sensitive files encrypted with age (`yadm config yadm.cipher age`), listed in `~/.config/yadm/encrypt`:
- `~/.ssh/id_ed25519`
- `~/.ssh/id_ed25519.pub`
- `~/.ssh/config`

Static config files tracked by yadm (not generated by scripts):
- `~/.zshrc`
- `~/.zprofile`
- `~/.gitconfig`
- `~/.local/bin/dsf-filter`
- `~/.config/hypr/hyprland.conf`
- `~/.config/hypr/hyprlock.conf`
- `~/.config/waybar/` (config.jsonc, style.css, power_menu.xml)
- `~/.config/yadm/encrypt`
- `~/.config/setup/` (the scripts themselves)

Do not put secrets in `~/.zshrc` / `~/.zprofile` ‚Äî they are tracked unencrypted. Use a separate sourced file added to `~/.config/yadm/encrypt` instead.

## Script responsibilities

### 1-post-install.sh
Runs pre-reboot from the archinstall environment. Takes `<device>` and `<luks-uuid>` as arguments.
1. Enrol TPM2 into the LUKS slot (`systemd-cryptenroll`)
2. Patch `/etc/default/grub` to add `rd.luks.name=UUID=<luks-uuid>=cryptroot`
3. `grub-mkconfig`
4. Rewrite HOOKS in `/etc/mkinitcpio.conf` to the systemd-based set, then `mkinitcpio -P`

### packages.sh
1. Install `base-devel` via pacman (needed to build yay)
2. Clone and build yay from AUR (`https://aur.archlinux.org/yay.git`)
3. `pacman -S` all pacman packages above
4. `yay -S` all AUR packages above
5. `rustup default stable`

### shell.sh
1. Install Oh My Zsh via upstream script (unattended, to `~/.oh-my-zsh`)
2. `chsh -s /usr/bin/zsh`

### neovim.sh
1. Clone `https://github.com/Encephala/kickstart.nvim` into `~/.config/nvim`
2. Run `nvim --headless "+Lazy! sync" +qa` to install plugins

### fonts.sh
1. Write `/etc/fonts/local.conf` with CaskaydiaMono Nerd Font as monospace default (sudo)
2. `fc-cache -f`

## Manual steps (printed by 2-bootstrap.sh at the end)

- **Hyprland local config**: create `~/.config/hypr/hyprland-local.conf`
  (monitor layout, cursor size, touchpad, brightness keys ‚Äî device-specific, gitignored)
- **Hyprlock local config**: create `~/.config/hypr/hyprlock-local.conf` (gitignored)
- **Hyprland GUI config**: `~/.config/hypr/hyprland-gui.conf` is managed by hyprmod, generated automatically
- **Synology Drive**: verify the service started; configure sync folders
- **Relog** for `chsh` to take effect

## Notes

- Oh My Zsh installed via upstream script (not AUR) so `~/.oh-my-zsh` path in `.zshrc` stays correct
- `dsf-filter` in `~/.local/bin/` is a thin wrapper required by `git config interactive.diffFilter`; `diff-so-fancy` itself is installed via pacman
- zoxide is installed via pacman (available in Arch repos), not cargo
- `~/.gitconfig` `signingkey` is hardcoded to `/home/jonathan/.ssh/id_ed25519.pub`
- starship provides the prompt; Oh My Zsh is used only for its plugin infrastructure (completions etc.)
